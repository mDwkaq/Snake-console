Основы контроля версий для одного разработчика
1. Проверить, что изменилось
bash

git status          # какие файлы изменены/добавлены
git diff            # что именно изменилось в файлах (пока не добавлено в stage)
git diff --staged   # что именно изменилось в файлах (уже после git add)

2. Откатить незакоммиченные изменения

Отменить изменения в одном файле (вернуть к последнему коммиту):
bash

git restore Snake.hpp          # новая команда (Git 2.23+)
git checkout -- Snake.hpp      # старая команда (работает везде)

Отменить все незакоммиченные изменения во всех файлах:
bash

git restore .

Если ты уже сделал git add, но ещё не коммитил:
bash

git restore --staged Snake.hpp  # убрать файл из stage (индекса)
git restore Snake.hpp           # затем откатить изменения

3. Отменить последний коммит (если ещё не запушил)

Самый безопасный способ – soft reset (изменения останутся в рабочей папке, но коммит исчезнет):
bash

git reset --soft HEAD~1

После этого ты снова можешь править файлы и сделать новый коммит.

Жёсткий сброс (все изменения удалятся, коммит исчезнет без следа):
bash

git reset --hard HEAD~1

Используй осторожно! Только если уверен, что последний коммит не нужен.

4. Откатить коммит, который уже запушил на GitHub
bash

git revert HEAD   # создаёт новый коммит, отменяющий изменения последнего
git push

Это безопасно для общего репозитория, история сохраняется.

5. Посмотреть историю и вернуться к старому состоянию
bash

git log --oneline       # компактный список коммитов (хэши и сообщения)
git show <хэш>          # посмотреть, что было в конкретном коммите
git checkout <хэш>      # временно переключиться на тот коммит (detached HEAD)
git checkout main       # вернуться обратно на ветку main

Для постоянного возврата лучше использовать reset или revert.

6. Экспериментальные ветки (чтобы не сломать main)
bash

git branch test-fix        # создать ветку
git checkout test-fix      # переключиться на неё
# или одной командой:
git checkout -b test-fix
# работаешь, коммитишь. Если не удалось:
git checkout main
git branch -D test-fix     # удалить ветку без сожаления
# Если удалось и хочешь перенести изменения в main:
git checkout main
git merge test-fix
git branch -d test-fix
