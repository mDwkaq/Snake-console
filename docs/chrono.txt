Основы <chrono> для игрового цикла (C++11)

Библиотека <chrono> работает с тремя понятиями:

    Часы (clock) — источник времени.

    Продолжительность (duration) — промежуток времени.

    Момент времени (time_point) — конкретная точка на временной шкале.

Какие часы использовать?

    std::chrono::steady_clock — монотонные часы. Идут только вперёд, не зависят от перевода системного времени. Идеальны для измерения интервалов в играх.

    std::chrono::system_clock — системные часы, могут скакать при коррекции времени. Не используем.

    std::chrono::high_resolution_clock — часто синоним одного из вышеуказанных, лучше не гадать, а брать steady_clock.

Как создать интервал (шаг игры)?

std::chrono::milliseconds — это тип duration, представляющий миллисекунды.
Пример:
cpp

std::chrono::milliseconds tickDuration(200); // 200 мс

В C++11 так и пишем. В C++14 можно было бы 200ms, но у тебя база C++11, поэтому запомни: всегда явно std::chrono::milliseconds(число).

Другие единицы: std::chrono::microseconds, std::chrono::seconds и т.д.

Как измерить прошедшее время?
cpp

using Clock = std::chrono::steady_clock;
auto previousTime = Clock::now(); // запомнили момент

// ... внутри цикла ...
auto currentTime = Clock::now();
auto elapsed = currentTime - previousTime; // elapsed — duration
if (elapsed >= tickDuration) {
    // пора делать шаг игры
    previousTime = currentTime; // сбросить таймер
    // ... игровая логика ...
}

elapsed — это тоже duration. Сравниваем с нашим tickDuration.

Как сделать паузу без Sleep?

Вместо Sleep(200) из <windows.h>, который блокирует поток и не даёт обрабатывать ввод, можно использовать:
cpp

std::this_thread::sleep_for(std::chrono::milliseconds(1));

Но в игровом цикле чаще делают иначе: либо крутят цикл с опросом ввода и проверкой времени, либо используют sleep
с очень маленьким интервалом (1-5 мс), чтобы не грузить процессор. Для консольной змейки хороший вариант: в каждой
итерации проверяем ввод, если время для шага ещё не пришло — делаем sleep_for(1ms) и продолжаем. Это даст отзывчивость на клавиши.

Как организовать игровой цикл (game loop)?

Типичная схема для змейки:
text

while (игра_не_закончена) {
    // 1. Обработать ввод (поменять направление)
    processInput();

    // 2. Проверить, не пора ли сделать игровой шаг
    auto now = Clock::now();
    if (now - lastUpdate >= tickDuration) {
        updateGame();      // движение, столкновения
        render();          // перерисовать экран
        lastUpdate = now;
    } else {
        // маленькая пауза, чтобы не нагружать CPU
        std::this_thread::sleep_for(std::chrono::milliseconds(1));
    }
}

Такой цикл позволяет змейке двигаться с постоянной скоростью (200 мс на клетку), а клавиши обрабатываются мгновенно.

Как это применить к твоей тестовой полоске

Ты хотел: цифра по умолчанию ползёт вправо, стрелки меняют направление. При достижении края — перескок на противоположную сторону.

Тебе понадобятся:

    Переменная direction (например, 1 для вправо, -1 для влево).

    Таймер с steady_clock.

    Цикл, в котором:

        Опрашиваются клавиши (можно _kbhit() + _getch() или GetAsyncKeyState). При нажатии стрелки меняем direction.

        Каждые 200 мс сдвигаем цифру: стираем старую, posX += direction, с учётом зацикливания, рисуем новую.

Чтобы ввод не блокировал таймер, нужно использовать неблокирующую проверку клавиш. Здесь мы отойдём от простого _getch()
и воспользуемся либо _kbhit() (проверяет, есть ли нажатие в буфере, и только тогда вызывает _getch()), 
либо GetAsyncKeyState (WinAPI, опрашивает состояние клавиши в данный момент). Для простоты и кроссплатформенности (в рамках Windows) можем взять GetAsyncKeyState — ты уже видел его в варианте 2.

. Как выглядит работа с <chrono> без auto

Допустим, мы хотим интервал 200 мс.

Шаг 1: выбираем часы
cpp

#include <chrono>
#include <thread> // для sleep_for

// Тип часов:
typedef std::chrono::steady_clock Clock;
// Тип момента времени:
typedef std::chrono::time_point<Clock> TimePoint;
// Тип интервала (duration) — 200 миллисекунд:
typedef std::chrono::milliseconds Milliseconds;

Milliseconds tickDuration(200); // интервал шага
TimePoint lastUpdate = Clock::now(); // запомнили начальный момент


Шаг 2: в игровом цикле
cpp

while (true) {
    // === Обработка ввода (без блокировки) ===
    if (GetAsyncKeyState(VK_LEFT) & 0x8000) {
        // меняем направление
    }
    // ... другие клавиши

    // === Проверка таймера ===
    TimePoint now = Clock::now();
    std::chrono::milliseconds elapsed = 
        std::chrono::duration_cast<std::chrono::milliseconds>(now - lastUpdate);
    
    if (elapsed >= tickDuration) {
        // Игровой шаг: стереть старую, обновить позицию, нарисовать
        lastUpdate = now; // сброс таймера
    } else {
        // маленькая пауза, чтобы не грузить процессор
        std::this_thread::sleep_for(std::chrono::milliseconds(1));
    }
}

Пояснение типов без auto:

    now - lastUpdate — это вычитание двух TimePoint, даёт duration в тиках часов (обычно наносекунды).

    std::chrono::duration_cast<std::chrono::milliseconds>(...) преобразует это в миллисекунды, чтобы сравнивать с tickDuration.

    lastUpdate = now; — присваивание момента.

Если бы мы использовали auto, то:
cpp

auto now = Clock::now();
auto elapsed = std::chrono::duration_cast<std::chrono::milliseconds>(now - lastUpdate);

Но без auto пишем явно, как выше.

. #include <thread> — зачем он нам?

<thread> даёт два полезных средства для игры:

    std::this_thread::sleep_for(duration) — усыпляет текущий поток на указанный промежуток времени.

    std::this_thread::sleep_until(time_point) — усыпляет до конкретного момента времени (на хай-рез таймерах удобно для стабильного шага, но для начала хватит sleep_for).

В нашем игровом цикле мы будем использовать sleep_for(1ms), чтобы не нагружать процессор пустым опросом клавиш, когда шаг игры ещё не наступил.

Важно: sleep_for — блокирующая операция для потока, но блокирует она его очень коротко (1 мс). Это допустимо, потому что мы сразу после пробуждения проверяем клавиши и таймер. GetAsyncKeyState тем временем не блокирует ничего — он просто мгновенно считывает физическое состояние клавиши.

    Почему GetAsyncKeyState не блокирует?
    Он лезет напрямую в состояние клавиатуры на уровне драйвера, минуя очередь сообщений консоли. Это как спросить: «Клавиша нажата прямо сейчас?» —
    и немедленно получить ответ. _getch() же ждала бы, пока в очереди появится символ (как cin.get()), поэтому её одной в игровом цикле недостаточно.

    Вот полный список литералов:

    h – часы (C++14, std::chrono::hours)

    min – минуты

    s – секунды

    ms – миллисекунды

    us – микросекунды

    ns – наносекунды

    Что ещё нового в chrono (C++17/20/23)

    C++17: std::chrono::ceil, floor, round для округления длительностей. Например, округлить наносекунды до миллисекунд вверх: std::chrono::ceil<milliseconds>(someNanos). В змейке не нужно, но пригодится для физики/сетей.

    C++20: std::chrono::clock_cast для конвертации между разными часами; добавили календарные типы (year_month_day), time_zone, но нам это не понадобится.

    C++23: мелочи, несущественные для игры.

Для наших целей достаточно литералов и sleep_for.
